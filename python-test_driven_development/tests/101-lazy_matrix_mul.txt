=================================
How to Use 101-lazy_matrix_mul.py
=================================

This document explains how to use the function ``lazy_matrix_mul(m_a, m_b)`` 
implemented in the accompanying script.

Usage
=====

``lazy_matrix_mul(m_a, m_b)`` returns a new matrix equal to the product of 
matrix ``m_a`` multiplied by matrix ``m_b``.

Example:
::

    >>> lazy_matrix_mul = __import__('101-lazy_matrix_mul').lazy_matrix_mul
    >>> m_a = [
    ... [1, 2],
    ... [3, 4],
    ... ]
    >>> m_b = m_a
    >>> print(lazy_matrix_mul(m_a, m_b))
    [[7, 10],
     [15, 22]]

::

    >>> m_a = [[1, 2]]
    >>> m_b = [
    ... [3, 4],
    ... [5, 6]
    ... ]
    >>> print(lazy_matrix_mul(m_a, m_b))
    [[13, 16]]

Floating point values are supported:
::

    >>> m_a = [
    ... [1.2, 5.5, 6.2],
    ... [4.66, 12.3, -9.2]
    ... ]
    >>> m_b = [
    ... [5.0, 3.3],
    ... [-2.9, 4.4],
    ... [7.2, 4.4]
    ... ]
    >>> print(lazy_matrix_mul(m_a, m_b))
    [[34.69, 55.44],
     [-78.61, 29.018]]

ValueErrors
===========

A ValueError occurs when the matrices cannot be multiplied:
::

    >>> m_a = [
    ... [1, 2],
    ... [3, 4],
    ... ]
    >>> m_b = [
    ... [1, 2],
    ... [2, 3],
    ... [4, 5]
    ... ]
    >>> lazy_matrix_mul(m_a, m_b)
    Traceback (most recent call last):
    ValueError: m_a and m_b can't be multiplied

Empty matrices are also invalid:
::

    >>> lazy_matrix_mul([[]], [[5, 6], [7, 8]])
    Traceback (most recent call last):
    ValueError: m_a and m_b can't be multiplied

::

    >>> lazy_matrix_mul([[5, 6], [7, 8]], [[]])
    Traceback (most recent call last):
    ValueError: m_a and m_b can't be multiplied

================

Only lists of lists containing integers/floats are allowed.

::
